## Overview

The AdvoLens backend exposes a REST API built with **FastAPI**. All endpoints return JSON unless otherwise noted.

- **Base URL (production):** `https://<your-render-url>`
- **Base URL (local dev):** `http://localhost:8000`
- **Interactive docs:** `GET /docs` (Swagger UI)
- **OpenAPI schema:** `GET /openapi.json`

### Authentication

Protected endpoints require a `Bearer` JWT token in the `Authorization` header:

```
Authorization: Bearer <access_token>
```

Tokens are obtained via `POST /auth/login`.

### Router Prefixes

| Router | Prefix | Description |
|--------|--------|-------------|
| Issues | `/issues` | Citizen issue CRUD + AI pipeline |
| Auth | `/auth` | Login, register, current user |
| Admin | `/admin` | Admin-only issue management |
| Notifications | `/notifications` | Citizen notification tracking |
| Analytics | `/analytics` | Hotspots, heatmap, CSV export |
| Engagement | `/issues` | Votes and comments (same prefix as issues) |

## Issues

### `POST /issues/`

Submit a new civic issue. Triggers the full AI analysis pipeline.

**Auth:** None (public)  
**Content-Type:** `multipart/form-data`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `image` | File | ✅ | Photo of the issue |
| `lat` | float | ✅ | GPS latitude (e.g. `10.0159`) |
| `lon` | float | ✅ | GPS longitude (e.g. `76.3419`) |
| `title` | string | ❌ | Optional title (auto-generated by Gemini if omitted) |
| `description` | string | ❌ | Optional description |

**Processing pipeline:**
1. Upload image to Cloudinary
2. Generate CLIP embedding (512-dim)
3. Gemini image analysis → caption + tags
4. Faiss similarity search (visual duplicate check, threshold=0.92)
5. PostGIS radius query (spatial duplicate check, 50m)
6. Auto-assign department via routing_service
7. Save to DB + add to Faiss index
8. Create in-app notification
9. Send email to department officials

**Response `200 OK`:**
```json
{
  "id": 42,
  "title": "Garbage pile on main road",
  "caption": "Large pile of plastic bottles and food waste on street corner",
  "tags": ["municipality", "garbage", "plastic_waste", "street_litter"],
  "status": "Open",
  "department": "municipality",
  "image_url": "https://res.cloudinary.com/your-cloud/image/upload/...",
  "lat": 10.0159,
  "lon": 76.3419,
  "tracking_token": "abc123xyz...",
  "duplicate_of": null,
  "created_at": "2025-03-15T10:30:00Z"
}
```

**Duplicate detected response:**
```json
{
  "id": 43,
  "duplicate_of": 42,
  "tracking_token": "def456uvw...",
  ...
}
```

### `PATCH /issues/{issue_id}/status`

Update an issue's status.

**Auth:** None (currently open — intended for admin use)

**Request Body:**
```json
{
  "status": "In Progress"
}
```

Valid statuses: `"Open"` | `"In Progress"` | `"Resolved"` | `"Closed"`

**Response `200 OK`:** Updated issue object.  
**Response `400 Bad Request`:** Invalid status value.  
**Response `404 Not Found`:** Issue not found.

**Side effects:**
- Creates in-app notification for the citizen
- Calls `notify_issue_resolved` / `notify_issue_reopened` helpers

## Admin

### `GET /admin/issues`

Get issues filtered by the current user's role.

**Auth:** Required  
**Role behavior:**
- `super_admin` → sees **all** issues (can filter by any department)
- `official` → sees **only their department's** issues

| Query Param | Type | Description |
|-------------|------|-------------|
| `status_filter` | string | Filter by status (`Open`, `In Progress`, etc.) |
| `department_filter` | string | Filter by department (super_admin only) |

**Response `200 OK`:** Array of issue objects, newest first.

---

### `PATCH /admin/{issue_id}/reassign`

**Auth:** Required — `super_admin` only

**Request Body:**
```json

  "department": "water_authority"
```

**Response `200 OK`:** Updated issue object.  
**Response `403 Forbidden`:** Not a super admin.  
**Response `404 Not Found`:** Issue not found.

## Notifications

All notification endpoints use the anonymous **citizen tracking token** (generated at issue submission and stored in the citizen's `localStorage`).

### `GET /notifications/my-notifications`

Get all notifications for a citizen.

**Auth:** None (uses token query parameter)

| Query Param | Type | Required | Description |
|-------------|------|----------|-------------|
| `token` | string | ✅ | Citizen tracking token |
| `limit` | int | ❌ | Max results (1–100, default 50) |

**Response `200 OK`:**
```json
[
  {
    "id": 1,
    "issue_id": 42,
    "type": "issue_created",
    "message": "Your report #42 has been submitted and assigned to Municipality",
    "is_read": false,
    "created_at": "2025-03-15T10:30:00Z"
  },
  {
    "id": 2,
    "issue_id": 42,
    "type": "status_updated",
    "message": "Your issue #42 is now being worked on. Status: In Progress",
    "is_read": true,
```

**Notification types:**

| Type | Trigger |
|------|---------|
| `issue_created` | Issue successfully submitted |
| `duplicate_detected` | Similar issue already exists |
| `status_updated` | Admin changes status |
| `issue_resolved` | Status changed to Resolved |

```json
    "eps_meters": 100,
    "min_samples": 3,

}
```

## Priority Score Algorithm

The `priority_score` field is auto-calculated and updated on every vote:

```
priority_score = (upvotes × 10) + status_bonus − age_penalty

Status bonus:
  Open        → +50
  In Progress → +25
  Resolved    → +0

Age penalty:
  − 1 per day old (capped at 30 days)

Minimum score: 0 (never negative)
```

**Example:** Issue with 5 upvotes, `Open` status, 3 days old:
```
= (5 × 10) + 50 − 3
= 50 + 50 − 3
= 97
```

## Error Responses

All error responses follow FastAPI's standard format:

```json
{
  "detail": "Human-readable error message"
}
```

| HTTP Code | Meaning |
|-----------|---------|
| `400` | Bad request (validation error, invalid data) |
| `401` | Unauthorized (missing or invalid token) |
| `403` | Forbidden (insufficient role) |
| `404` | Resource not found |
| `500` | Internal server error (ML pipeline failure, DB error) |
